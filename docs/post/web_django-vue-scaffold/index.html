工具安装与vue项目创建

  1. chrome 浏览器 (页面测试)

  2. 打开你的音乐播放器 (可选)

  3. 安装 NodeJS (根据自己的OS选择)

    1. 命令行测试 node 运行

    2. npm 配置换源, 建议使用淘宝源

      npm config set registry https://registry.npm.taobao.org

      npm config get registry

  4. 安装 vue [读作 view]

    1. npm install -g vue-cli
    2. 安装完成 vue -v

  5. 安装 vue IDE

    1. jetbrains : WebStom
    2. ms: VSCode
      1. 插件 Vetur
      2. 插件 vscode-VueHelper
    3. atom
      1. 插件 : language-vue
    4. DCloud: HBuilder

创建 vue 脚手架

  1. cd <你的工作目录>
  2. vue init webpack <项目名称> (项目名称请使用英文小写)
  3. 在交互式命令行中输入项目的配置
    1. 项目名称
    2. 描述
    3. 作者
    4. 是否安装 vue-router [y]
    5. …
    6. 是否使用 npm [y]
    7. …
  4. IDE 编辑器打开 vue 项目目录
  5. 终端进入项目工作目录 npm run dev 启动项目

配置你的页面

  1. 编辑 src/components/xxx.vue 也就是页面模板
  2. 文档参考 vue 介绍

路由配置

  1. 添加页面 xxx.vue 到 src/components/

  2. 编辑 src/router/index.js 配置 url

    // 在开头引入模块
import <页面模块> from "@/components/<你的vue组建>"
   
// 在 routers 段落增加
   
{
path: "</url路径>",
name: "<路由名称>",
components: <模块>
}

  3. 增加 router 链接到页面 这里 src/App.vue 增加 root 和 自定义的 url

    <div id="app">
...
    <router-link to="/">Root<router-link>
    <router-link to="/url路径">路由名称<router-link>
...
</div>

    你的 vue 应用配置应该在页面已经更新了.

搭建一个可访问的后端

这里的步骤仅作快速导览, 具体操作疑问请参考官方文档或结合搜索引擎解决

安装 python 开发环境

  1. 推荐 python 3.7+

  2. 安装 pycharm (可选)

  3. 配置 pip 使用国内源 (例如清华/阿里)

  4. 安装 django

    直接安装使用 pip install django -i https://pypi.mirrors.ustc.edu.cn/simple/

创建一个 django 项目

使用 IDE 创建或者手动均可

  1. 项目工作目录
  2. 虚拟环境
  3. 项目模板语言 Django
  4. 应用名称 <自定义即可>
  5. … 其他配置步骤

启动 django 项目

  1. 执行迁移数据

    python manage.py migrate

    配置 settings.py 允许访问的主机

    ALLOWED_HOSTS = ['*']

    如果 django 版本不是新版本, 记得在 settings.py 手动注释 MIDDLEWARE 中的

    'django.middleware.csrf.CsrfResponseMiddleware',

    语言: zh-hans

    时区: Asia/Shanghai

  2. 运行 django 服务器

    python manage.py runserver

添加 url

  1. 项目 url 添加应用 url 的引用

  2. 配置应用的 url

    编辑 <项目目录>/<你的应用>/urls.py

    # 以应用名为 mysite 为例

path('mysite/', include('mysite.urls'))

写一个测试 api

配置 url 下的 views 视图配置

  1. 在 <项目目录>/<应用>/views.py 中定义后续用到的 api

    # 这里写一个叫 test_api 的视图
from django.http import JsonResponse
   
def test_api(request):
    return JsonResponse({"status": 0, "message": "Hello, this is a response msg form django test api."})

  2. 检查 web 服务 url 的页面返回内容

    格式是 <服务地址>/url/ 这里测试环境对应 http://127.0.0.1:8000/mysite/

跨域请求配置

在发起请求的处理上我们需要用到 axios 库, 在 vue.js 2.0 也推荐使用 axios 来完成 ajax 请求

npm install axios --save

配置 src/router/main.js 引入模块

import axios from 'axios'

Vue.prototype.$axios = axios

这样就可以使用 axios 来向 django 发起请求了, 把获取请求的动作放在 created 作用域, 我们的 vue 配置看起来是这样的

created() {
    this.$axios.get("http://127.0.0.1:8000/mysite/")
    	.then(response =>{
    	console.log(response.data)
    });
}

但这样并不会生效, 反而是在控制台我们会看到访问会被阻止的报错, 因为这样的访问是跨域的.

所以我们还需要处理一下访问跨域的问题, 跨域访问的解决方法会有很多, 我们这里采用的是通过 webpack 配置 vue 的 proxyTable 来解, 在 <项目目录>/config/index.js 中 ProxyTable 中的具体配置如下

module.exports = {
  dev: {

    // Paths
    assetsSubDirectory: 'static',
    assetsPublicPath: '/',
    proxyTable: {
      '/site': {
        // 测试环境
        target: 'http://127.0.0.1:8000/mysite/',  // 接口域名
        changeOrigin: true,  //是否跨域
        pathRewrite: {
          '^/site': ''   //需要rewrite重写的,
        }
      }
    },

配置完跨域配置, 我们也要调整一下 vue 的配置信息, 假设我们要绑定到 django_message 这个变量名上, 例如

<template>
  <div id="User">
    {{ django_message }}
  </div>
</template>

<script>
    export default {
        name: "User",
        data() {
          return {django_message: ""
          }
        },
        created() {
          this.$axios.get("/site/")
            .then(response =>{
              this.django_message = response.data.message
            })
        }
    }
</script>

此时如果一切配置无误, 那么我们请求数据就被拿到并且显示到页面上了, 我们也就有了一个可以上手开发环境了.

后续 django 就可以:

  • 创建超级管理员
  • 模型与数据库交互
  • 为vue序列化json数据

然后就可以进行应用的开发了.
